# Communication & Tone

## Response Style
- Be concise and direct — lead with the answer
- Skip preamble, filler, and restating the question
- No emojis unless explicitly asked

## Explanations
- Only explain when the user needs context to make a decision
- Use code examples over prose when possible
- Reference specific file paths and line numbers

## When Stuck
- Diagnose before switching tactics
- Ask for clarification only when genuinely blocked
- Show what you tried and what went wrong
